# aigzero Feedback

This repository is the central intake space for aigzero user feedback, bug reports, product issues, and small improvement requests.

It exists to keep feedback visible, searchable, and triageable outside the implementation repositories. The implementation work may happen in `aigzero-core`, `aigzero-web`, `aigzero-website`, `aigzero-docs`, or another project repository, but the original user-facing signal should remain easy to find here.

## What to Report Here

Use this repository for:

- Bug reports from the public product experience.
- User feedback about confusing, missing, or unreliable behavior.
- UX(User Experience) issues that affect task completion.
- Small product improvement requests that come from real usage.
- Follow-up notes from support, testing, or manual review.

## What Not to Include

Do not post secrets, API keys, private documents, sensitive user data, or full production logs. If a report depends on private material, describe the symptom, affected area, and a safe reproduction path instead.

Security-sensitive reports should not be filed as public issues. Use the private security reporting channel configured for the organization when available.

## Triage Expectations

Each issue should make the user impact clear before implementation details:

- What happened?
- What did the user expect?
- Where did it happen?
- How can the team reproduce or inspect it?
- Is there a screenshot, URL, task id, or browser console detail that can be shared safely?

When a feedback item becomes implementation work, link the corresponding pull request or issue in the target repository so the feedback trail remains connected to the fix.
